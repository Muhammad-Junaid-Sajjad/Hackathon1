# Add Theory Content Skill

Add theoretical explanations and concept definitions to a section.

## Arguments
- `$ARGUMENTS` - Section path and concepts (e.g., "M1/C2/S3 - ROS nodes, topics, services")

## Instructions

For each concept in the section, add:

### 1. Definition Block
```markdown
### What Is [Concept]?

**[Concept]** is [one clear sentence definition].

[1-2 paragraphs expanding with context]
```

### 2. Motivation Block
```markdown
### Why Do We Need [Concept]?

**Without [Concept]:**
- [Problem 1]
- [Problem 2]

**With [Concept]:**
- [Benefit 1]
- [Benefit 2]
```

### 3. Working Principle Block
```markdown
### How Does [Concept] Work?

[Step-by-step explanation]

1. First, [step 1]
2. Then, [step 2]
3. Finally, [step 3]
```

### 4. Analogy
Add a relatable real-world analogy:
```markdown
:::tip Real-World Analogy
Think of [concept] like [everyday analogy]. Just as [familiar thing] does [action], [concept] does [similar action] in robotics.
:::
```

### Guidelines
- Define terms BEFORE using them
- Use simple language first, then technical terms
- Connect to Physical AI and robotics applications
- Avoid jargon without explanation
